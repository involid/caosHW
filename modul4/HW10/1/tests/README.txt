Ожидаемое имя программы:
mergefs.c

Для программы есть два теста
Они запускаются скриптами run_1.sh и run_2.sh
Чтобы удалить временные файлы и саму файловую систему,
созданные в результате тестов,
используются скрипты clear_1.sh и clear_2.sh

Для запуска 2-го теста после 1-го, и наоборот,
нет необходимости чистить что-то за предыдущим тестом
Но для перезапуска одного и того же теста чистка обязательна

После запуска тестов остаются временные файлы, помогающие понять ошибку
В основной директории есть файл cmake_make_output_1.txt/cmake_make_output_2.txt,
содержащий вывод сборки cmake и make
Появляется директория mnt_1/mnt_2, содержащая саму примонтированную файловую систему
Её можно сравнить с тем, что ожидалось
В директории для тестов dirs_test_1/dirs_test_2 есть поддиректория expected_file_system
Она и демонстрирует, как должна выглядеть данная файловая система, с той разницей,
что там лежат обычные файлы
Корректность файловой системы проверяется с помощью файлов, лежащих
в expected_output и создаваемой директории real_output

В той же директории для тестов в поддиректории dirs лежат директории обычной файловой системы,
по которым и строится mergefs
В скриптах run_1.sh и run_2.sh можно посмотреть, как они передаются
Эта директория dirs также создаётся в момент тестов, создающие скрипты можно увидеть и
посмотреть в директории scripts
Там же есть полезный файл, описывающий, как выглядит структура данной поддиректории dirs,
с названием dirs_description.txt

Скрипты в директории scripts:
- dirs_create.sh: создаёт директорию dirs
- dirs_clean.sh: чистит директорию dirs
- check_create.sh: проверяет созданную файловую систему
- check_clean.sh: чистит временные файлы после проверки
